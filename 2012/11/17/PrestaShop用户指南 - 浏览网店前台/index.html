PrestaShop用户指南 - 浏览网店前台

发表于 2012-11-17 分类于 程序人生 阅读次数： 本文字数： 6k 阅读时长 ≈ 5 分钟

浏览网店前台----翻译累死人，看看后台，modules->position->LiveEdit一下子就都能看到了，其实关键还是说后面那些东西来控制这些显示的～

前台是顾客访问你的网店的入口，包括了界面、图片、商品、商品描述以及整个购买的流程。 作为顾客，前台是你在浏览和购买时候的对于整个网店的体验。 作为网店店主，你也需要前台非常了解，不仅仅是因为网店是你的，更因为你需要知道你的顾客面对的是什么。

默认主题

PrestaShop拥有一个默认的主题，前端为灰，背景为白。这是国际通用的简单直观的设计。如果你安装了例子数据，你可以看到苹果的产品。 基于PrestaShop Addons网站的贡献，你可以在任何时候非常方便的改变前台的主题。这里，我们只是基于默认主题做介绍。

在线商店导航

不论客户是输入网店地址还是通过搜索引擎，他们总有许多选项通过下面的页面分类来导航。

顶部

顶部是一个狭窄的区域，可以在任何一个前台页面中访问。 这里包含了几个重要的工具和连接：
  • 网店Logo. 点击此LOGO可以回到首页面。请用你自己的Logo来代替我们的默认的“YourLogoHere”（默认大小是209x52）。
  • 搜索引擎. 许多顾客愿意通过搜索引擎而不是通过商品类别导航来查找指定的商品。
  • 联系和网站地图. 这两个连接让客户访问指定的页面，一个是网店管理员，一个完整的网站的公开的页面清单。
  • 添加收藏. 这不是一个连接而是一个JavaScript行为，当点击之后，将让顾客的浏览器把你的网店加入收藏夹。
  • 货币选项. 顾客可以选择商品标价的货币。
  • 购物车: (空). 是一个关于顾客当前购物车状态的快速提醒。
  • 欢迎. 简单的欢迎信息，当顾客登录之后，会变成“欢迎，姓”然后这个“姓”会连接到客户的帐号。
  • 登录. 是邀请客户注册或者登录的连接。
在整个购物过程中，顶部基本上是不变的。

购物车

顶部最重要的部分是购物车。默认地，这是用来显示当前已经包含的商品的数量。当客户点击的时候，可以访问购物车，并开始购物的核对流程。 当顾客把鼠标指向购物车的时候，这个区域会改变成显示每个增加的商品的名字。每个商品行包含了数量，组合和价格。顾客能够直接点击移除不想要的商品。在这个区域的底部会有配送费用和此订单的总费用。当点击核对的时候，可以开始核对流程。

底部

底部提供了一些对于客户来说是有用的页面。
  • ”我的帐号“区域, 包含了客户的个人资料的页面
    • 我的订单. 所有过去和现在在处理的订单。
    • 我的存款. 可以用于将来订单的存款。
    • 我的地址. 客户可以增加多个不同的地址。
    • 我的个人信息. 姓名、电邮、家庭地址、电话号码、出生日期等所有必需的信息。
    • 我的优惠券. 所有还没有被使用的优惠券号码。
    • 我关注的商品. 客户可以标记一些商品，以便今后购买。
  • “信息”区域
    • 三个和商品有关：
      • 促销. 所有当前的促销。
      • 新品. 近来增加的商品。
      • 热销品. 当前热卖的商品。
    • 四个是基本信息:
      • 我们的网店
      • 联系我们
      • 服务条款
      • 关于我们
  • “关注我们”区域,包含了你的网店的Facebook/Twitter帐号，并联系到你的RSS种子。
  • “联系我们”区域,包含了网店的地址，联系电话，电邮等。
所有这些内容店主都能改变。

左列

默认主题的左列主要是静态的，主要是用来导航。

-1.标签区

店主可以为每个商品标注一系列的标签。标签是没有继承关系的关键字。这些不会再商品页面被显示，但是在建主题清单的时候会非常有用。另外，大部分标签用粗体来显示。

-2.商品类别区

商品类别是有继承关系的显示商品的方式，能包含任何数量的子类别，能方便地随着逻辑关系找到指定的商品。 PrestaShop网店可以设置足够多的类别和子类别，在指定的类别层，可以设置无限多的商品。所有的类别都是根类别“HOME”的子类。 这也是唯一的客户可以通过选择来比较商品的商品清单。

-3."看过的商品" 区

显示在当前Session中客户至少看过一次的商品。 这是对于客户看过的商品的提醒，同时提供了返回客户看过的商品的相信页面的快捷方式。

-4.供应商区

客户可以选择访问所有来自同一个供应商的商品。

-5.制造商区域

客户可以选者访问所有来自同一个制造商的商品。Customers can choose to display all the products from a single manufacturer, regardless of their types or prices.

-6.CMS区域

这里PrestaShop列出了所有由店主创建的静态页面：配送信息、法律声明、服务条款等。

-7.广告区域

默认这里只是一个简单的连接到PrestaShop官方主页的图片。你可以通过改变设置，把这里变成其他网站的广告连接。

-8.新闻区域

客户可以直接用此模块定制接收你的网店的更新新闻。

右列

默认主题的右列在购物过程中基本是不变的。

-1.新品区域

此处显示所有的新品，并连接到他们的详细页面。

-2.促销区域

显示所有的促销商品。

-3."我们的商店"区域

此处包含了一个到商店地理定位的图片连接，可以让你的客户找到你的最近的实体店。当然如果你没有实体店的话，也可以通过设置关闭。

-4."联系我们" 区域

此处显示你的商店的服务热线。

中列

这是根据客户的需要会变化的区域。

-1.前端页面

默认的前端页面是给客户介绍商店的。 紧跟这的就是“首推商品” 大部分用户都是通过搜索引擎来访问网店，直接浏览商品或者类别。首页必须有特色，能够吸引新客户。

-2.产品列表页

PrestaShop有许多方式让客户找到商品，比如类别，标签，制造商，供应商，搜索等等，但是最终是需要给客户一个商品列表的。 尽管内容可能不同，列表被设计成相近的显示方式：
  • 左边是个选择框
  • 中间是图片，名字和说明
  • 右边是价格，库存及“添加到购物车”
ategory header: image and image map
Categories can feature a header image with an introductory message, as set by the shop owner. While this is not the case by default, you can choose to have the category image replaced by an image map. A category can feature an unlimited number of image maps, which the visitor can scroll through with the tool below it. An image map is simply an image which has links available for some of its areas. A "+" symbol is placed upon the areas which have links. The image map can feature as many links as necessary.
Product sorting
Product listing by category or manufacturer can be further sorted by price, name or availability. Note that listing by tags does not allow for such sorting.
Product comparison
Products within a category can be compared to each other, thanks to each item′s checkbox (on the left of the thumbnail). This is the only product listing where the customer can compare products, since there is no sense in comparing products that do not have the same characteristics/features. Obviously, this means that categories should only feature products which can be compared. Provided the product′s information is complete and consistent, the comparison page will display with a row for each comparable feature. This is immensely useful, particularly for products with technical differences. By default, customers can compare up to 3 products at a time, but this can be changed in the "Products" preference page in the back-office.

Product page

This is where all the information entered by the shop owner is available to the user. Depending on the theme design, a product page can be very thorough, with extensive information, or simply present the most essential facts. The default theme is typical in that its most prominent feature is the product images, with a tool below it enabling customers to scroll between the available images. Next to the images are two blocks:
  • A "Short description" block, presenting the main facts for the current product.
  • The "Add to cart" block, with the option to choose among the available combinations (as defined by the shop owner) and the quantity to be ordered.
At the bottom of the product page is a tab section. The most usual ones are:
  • More info. This gives the full description for the product, as entered by the shop owner.
  • Data sheet. This tab only appears if the shop owner has entered data in the product′s "Features" sheet. This gives all the detailed featured that were entered into database. This data is also the one that is used when comparing two products. Therefore, the content of this tab is very sparsely written: these are just raw facts, far from the product description in the "More info" tab.

The cart page

Clicking on the "Cart: XX products" link in the header, or on the "Check out" button in the cart interface also in the header, brings the customer to the "Shopping cart summary" page, which is the first step in the order process. The breadcrumb trail at the top of the page indicates the steps in the order process:
  1. Summary. Where the customer makes sure that the order only contains the wanted products, no more, no less.
  2. Login. Where the non-logged customer is asked to log in or create an account. This step is skipped over if the customer is already logged in.
  3. Address. Where the customer is presented with his or her currently registered addresses in PrestaShop, and has to choose the one where the delivery is to be made.
  4. Shipping. Where the customer choose the shipping options.
  5. Payment. Where the customer is presented with the final price of the order (now including the shipping price), and is asked to choose a payment method.
The customer can come back to any previous step by clicking on its title.
The standard order process takes 5 steps, and as many pages. You can choose to have all these steps on one page by going in the back-office, and in the "Orders" preference page, change the "Order process type" option from "Standard (5 steps)" to "One page check out".
The cart page gives a succinct but complete view of the items that are in the cart: thumbnail, name, features, unit price and ability to change the quantity of each product. Detailed pricing is then displayed below, including the order′s total cost with tax and shipping, and if applicable, a description of the voucher that was used. Further below, the available addresses are displayed. Clicking "Next" starts the check-out process.
